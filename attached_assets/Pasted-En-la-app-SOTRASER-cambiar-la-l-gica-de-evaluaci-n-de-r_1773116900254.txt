═══════════════════════════════════════════════════
LÓGICA NUEVA — PERCENTIL 90 COMO REFERENCIA
═══════════════════════════════════════════════════

En lugar de comparar cada camión contra una meta 
fija de 2.1 km/L, calcular el percentil 90 
del rendimiento real de toda la flota 
desde 2026-03-01.

Cálculo en el servidor:

SELECT PERCENTILE_CONT(0.90) 
WITHIN GROUP (ORDER BY rendimiento_kmL) 
AS p90,
PERCENTILE_CONT(0.75) 
WITHIN GROUP (ORDER BY rendimiento_kmL) 
AS p75,
PERCENTILE_CONT(0.50) 
WITHIN GROUP (ORDER BY rendimiento_kmL) 
AS p50
FROM volvo_ecu
WHERE periodo_inicio >= '2026-03-01'
AND rendimiento_kmL > 0
AND rendimiento_kmL < 15;

Usar estos percentiles como umbrales dinámicos:
- EXCELENTE  = rendimiento >= P90  → verde brillante
- BUENO      = rendimiento >= P75  → verde suave
- NORMAL     = rendimiento >= P50  → azul
- BAJO       = rendimiento < P50   → amarillo
- CRÍTICO    = rendimiento < P50 * 0.7 → rojo

Endpoint nuevo: GET /api/flota/percentiles
Devuelve: { p50, p75, p90, total_camiones, 
            fecha_calculo }

Recalcular percentiles cada 6 horas y 
guardar en tabla parametros:
INSERT INTO parametros VALUES
('percentil_90', valor, NOW()),
('percentil_75', valor, NOW()),
('percentil_50', valor, NOW())
ON CONFLICT(clave) DO UPDATE SET 
valor = EXCLUDED.valor,
updated_at = NOW();

═══════════════════════════════════════════════════
CASOS ESPECIALES
═══════════════════════════════════════════════════

Si hay menos de 10 camiones con datos suficientes
para calcular percentiles confiables:
- Usar meta fija temporal de 2.1 km/L
- Mostrar aviso: "Percentiles en calibración — 
  se necesitan más datos desde 01-03-2026"

Excluir del cálculo de percentiles:
- Rendimientos > 15 km/L (imposibles, datos corruptos)
- Rendimientos = 0 (sin datos)
- Camiones con menos de 3 registros desde 01-03-2026
