CREATE TABLE IF NOT EXISTS puntos_ruta (
  id SERIAL PRIMARY KEY,
  camion_id INTEGER REFERENCES camiones(id),
  lat NUMERIC(10,7) NOT NULL,
  lng NUMERIC(10,7) NOT NULL,
  tipo VARCHAR(20),
  nombre_inferido VARCHAR(200),
  duracion_min INTEGER,
  fecha TIMESTAMP,
  confirmado BOOLEAN DEFAULT false,
  confirmado_por VARCHAR(80),
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS tarifas_ruta (
  id SERIAL PRIMARY KEY,
  nombre_ruta VARCHAR(200) NOT NULL,
  punto_origen_id INTEGER REFERENCES puntos_ruta(id),
  punto_destino_id INTEGER REFERENCES puntos_ruta(id),
  origen_nombre VARCHAR(200),
  destino_nombre VARCHAR(200),
  distancia_km NUMERIC(8,1),
  litros_promedio NUMERIC(8,2),
  tiempo_horas NUMERIC(5,2),
  tarifa_clp NUMERIC(12,0),
  tarifa_usd NUMERIC(10,2),
  notas TEXT,
  activa BOOLEAN DEFAULT true,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE INDEX IF NOT EXISTS idx_puntos_camion 
  ON puntos_ruta(camion_id);
CREATE INDEX IF NOT EXISTS idx_tarifas_activa 
  ON tarifas_ruta(activa);

BLOQUE 6 — PESTAÑA NUEVA: TARIFAS
═══════════════════════════════════════════════════

Agregar pestaña nueva "TARIFAS" al final de todas,
DESPUÉS de CONFIGURACION. Completamente separada
del resto de la app. No afecta ningún cálculo existente.

Esta pestaña tiene 2 sub-secciones con tabs internos:
[PUNTOS CONFIRMADOS] [TARIFAS POR RUTA]

─── SUB-SECCIÓN: PUNTOS CONFIRMADOS ───

Muestra todos los puntos de puntos_ruta 
donde confirmado = true, agrupados por tipo.

Endpoint GET /api/tarifas/puntos-confirmados:
SELECT * FROM puntos_ruta 
WHERE confirmado = true 
ORDER BY tipo, nombre_inferido

Tabla con columnas:
TIPO · NOMBRE · COORDENADAS · CAMIONES QUE PASAN · 
FECHA PRIMERA VEZ · ACCIONES

Botón "✏ Editar nombre" para corregir el nombre
que detectó la IA.

Filtro por tipo: TODOS / CARGA / ENTREGA / 
COMBUSTIBLE / DESCANSO

─── SUB-SECCIÓN: TARIFAS POR RUTA ───

Permite crear tarifas entre pares de puntos confirmados.

Endpoint GET /api/tarifas:
SELECT t.*, 
  po.nombre_inferido AS origen,
  pd.nombre_inferido AS destino
FROM tarifas_ruta t
LEFT JOIN puntos_ruta po ON po.id = t.punto_origen_id
LEFT JOIN puntos_ruta pd ON pd.id = t.punto_destino_id
WHERE t.activa = true
ORDER BY t.nombre_ruta

Endpoint POST /api/tarifas: crear nueva tarifa
Endpoint PUT /api/tarifas/:id: editar tarifa
Endpoint DELETE /api/tarifas/:id: desactivar tarifa

Formulario crear tarifa:
- Nombre de la ruta (texto libre)
  Ej: "Santiago CD Vespucio → La Serena"
- Origen: dropdown de puntos confirmados tipo CARGA
- Destino: dropdown de puntos confirmados tipo ENTREGA
- Distancia km: pre-llenado automático si hay datos GPS,
  editable manualmente
- Litros promedio: pre-llenado con promedio histórico
  de camiones que hicieron esa ruta, editable
- Tiempo estimado horas: editable
- Tarifa CLP: campo principal editable
- Tarifa USD: campo opcional editable
- Notas: texto libre

Tabla de tarifas existentes:
RUTA · ORIGEN · DESTINO · KM · LITROS PROM · 
TIEMPO · TARIFA CLP · TARIFA USD · ACCIONES

Botones por fila: Editar · Desactivar

KPIs arriba de la tabla:
- Total rutas activas
- Ruta más cara CLP
- Ruta más frecuente (más camiones)
- Promedio litros/ruta

IMPORTANTE: Esta pestaña es solo gestión de 
tarifas y puntos. No afecta ningún cálculo 
de rendimiento, alertas ni cuadratura.
Es completamente independiente del resto.
