Agregar inteligencia de detección automática

-- Lugares conocidos detectados automáticamente
CREATE TABLE IF NOT EXISTS geo_lugares (
  id SERIAL PRIMARY KEY,
  nombre VARCHAR(200),
  nombre_confirmado VARCHAR(200),
  tipo VARCHAR(50),
  -- CD_CENCOSUD · LOCAL_JUMBO · LOCAL_SANTA_ISABEL
  -- LOCAL_CENCOSUD · BASE_ORIGEN · SURTIDOR · OTRO
  lat NUMERIC(10,7) NOT NULL,
  lng NUMERIC(10,7) NOT NULL,
  radio_metros INTEGER DEFAULT 500,
  direccion VARCHAR(300),
  comuna VARCHAR(100),
  region VARCHAR(100),
  -- Detección automática
  detectado_via VARCHAR(50),
  -- OSM · GOOGLE · COMPORTAMIENTO · MANUAL
  confianza_pct INTEGER DEFAULT 0,
  veces_visitado INTEGER DEFAULT 0,
  primera_visita DATE,
  ultima_visita DATE,
  -- Control
  confirmado BOOLEAN DEFAULT false,
  activo BOOLEAN DEFAULT true,
  creado_at TIMESTAMP DEFAULT NOW()
);

-- Visitas de camiones a lugares
CREATE TABLE IF NOT EXISTS geo_visitas (
  id SERIAL PRIMARY KEY,
  lugar_id INTEGER REFERENCES geo_lugares(id),
  camion_id INTEGER REFERENCES camiones(id),
  patente VARCHAR(20),
  viaje_id INTEGER REFERENCES geo_viajes(id),
  llegada TIMESTAMP,
  salida TIMESTAMP,
  minutos_detenido INTEGER,
  lat_exacta NUMERIC(10,7),
  lng_exacta NUMERIC(10,7),
  creado_at TIMESTAMP DEFAULT NOW()
);

-- Historial de análisis IA
CREATE TABLE IF NOT EXISTS geo_analisis_ia (
  id SERIAL PRIMARY KEY,
  tipo VARCHAR(50),
  -- LUGARES · RUTAS · PATRONES · ANOMALIAS
  periodo_desde DATE,
  periodo_hasta DATE,
  resultado_json JSONB,
  resumen_texto TEXT,
  generado_at TIMESTAMP DEFAULT NOW()
);

-- Agregar columnas a geo_viajes si no existen

  ADD COLUMN IF NOT EXISTS

SERVICIO — DETECCIÓN AUTOMÁTICA DE LUGARES
═══════════════════════════════════════════════════

Función detectarLugar(lat, lng, minutos_detenido):

Corre cada vez que un camión lleva
> 60 minutos detenido en un punto.

PASO 1 — ¿Ya conocemos este lugar?
Buscar en geo_lugares dentro de radio_metros:
SELECT * FROM geo_lugares
WHERE earth_distance(
  ll_to_earth(lat, lng),
  ll_to_earth($lat, $lng)
) < radio_metros
ORDER BY earth_distance(...) ASC
LIMIT 1;

Si hay match → es un lugar conocido.
Actualizar veces_visitado y ultima_visita.
Retornar lugar existente.

PASO 2 — Lugar nuevo, identificar qué es
Si no hay match → consultar OpenStreetMap
Nominatim para reverse geocoding:

GET https://nominatim.openstreetmap.org/reverse
  ?lat={lat}&lon={lng}&format=json

Y consultar Overpass API para buscar
negocios CENCOSUD cercanos en radio 500m:

Query Overpass:
[out:json];
(
  node["name"~"Jumbo|Santa Isabel|Cencosud|
       Easy|Paris|Johnson|Disco",i]
  (around:500,{lat},{lng});
  way["name"~"Jumbo|Santa Isabel|Cencosud|
      Easy|Paris|Johnson|Disco",i]
  (around:500,{lat},{lng});
);
out body;

URL: https://overpass-api.de/api/interpreter
  ?data={query_encoded}

PASO 3 — Clasificar el lugar según resultado:

Si Overpass encuentra "Jumbo":
  tipo = 'LOCAL_JUMBO'
  nombre = 'Jumbo ' + dirección
  confianza = 90

Si encuentra "Santa Isabel":
  tipo = 'LOCAL_SANTA_ISABEL'
  nombre = 'Santa Isabel ' + dirección
  confianza = 90

Si encuentra "Cencosud" genérico:
  tipo = 'LOCAL_CENCOSUD'
  nombre = 'Local Cencosud ' + dirección
  confianza = 85

Si encuentra "Easy":
  tipo = 'LOCAL_EASY'
  nombre = 'Easy ' + dirección
  confianza = 85

Si encuentra "Paris" o "Johnson":
  tipo = 'LOCAL_PARIS'
  nombre = 'Paris/Johnson ' + dirección
  confianza = 85

Si no encuentra nada de CENCOSUD pero
el camión lleva > 2 horas:
  tipo = 'PUNTO_FRECUENTE'
  nombre = geocoding reverso (ciudad, calle)
  confianza = 50
  -- Marcar para revisión manual

PASO 4 — Guardar lugar nuevo:
INSERT INTO geo_lugares con todos los datos.
Retornar lugar creado.

PASO 5 — Registrar visita:
INSERT INTO geo_visitas con:
lugar_id, camion_id, patente, viaje_id,
llegada, lat_exacta, lng_exacta.

ANÁLISIS HISTÓRICO DESDE 01-03-2026
═══════════════════════════════════════════════════

Función analizarHistoricoCompleto():
Corre UNA VEZ al instalar y bajo demanda.

PASO 1 — Obtener todas las paradas históricas:
SELECT DISTINCT ON (DATE_TRUNC('hour', timestamp_punto),
  ROUND(lat::numeric, 3),
  ROUND(lng::numeric, 3))
  camion_id, patente, lat, lng,
  timestamp_punto, velocidad_kmh
FROM geo_puntos
WHERE timestamp_punto >= '2026-03-01'
AND velocidad_kmh < 5
ORDER BY DATE_TRUNC('hour', timestamp_punto),
  ROUND(lat::numeric, 3),
  ROUND(lng::numeric, 3),
  timestamp_punto;

PASO 2 — Agrupar paradas consecutivas
por camión en mismo lugar (radio 300m):
Para cada grupo de puntos estáticos
consecutivos del mismo camión calcular:
- Tiempo total detenido (min)
- Coordenada central del grupo (promedio lat/lng)
- Timestamp inicio y fin

PASO 3 — Filtrar paradas > 60 minutos:
Para cada parada > 60 min →
llamar detectarLugar(lat, lng, minutos)

PASO 4 — Reconstruir viajes históricos:
Para cada camión desde 2026-03-01:
Usando los puntos GPS, detectar viajes
según las reglas de detección (salida de base,
movimiento, llegada a destino).
Guardar en geo_viajes si no existe ya.

PASO 5 — Validar viajes históricos:
Para cada viaje histórico creado →
llamar validarConSigetra(viaje_id)

Mostrar progreso en pantalla:
"Analizando historial desde 01-03-2026...
 Camión 1/14: DKPW78 → 47 puntos procesados
 Lugares detectados: 8
 Viajes reconstruidos: 23"
