FUENTES DE DATOS REALES — leer y entender primero
═══════════════════════════════════════════════════

ANTES DE ESCRIBIR CÓDIGO hacer estas queries
para entender exactamente qué hay en la BD:

-- ¿Qué tablas GPS existen y cuántos registros?
SELECT 'gps_registros' as tabla,
  COUNT(*) as total,
  MIN(timestamp_gps) as desde,
  MAX(timestamp_gps) as hasta
FROM gps_registros
WHERE timestamp_gps >= '2026-03-01'
UNION ALL
SELECT 'volvo_ecu',
  COUNT(*),
  MIN(periodo_inicio),
  MAX(periodo_inicio)
FROM volvo_ecu
WHERE periodo_inicio >= '2026-03-01';

-- ¿Qué campos tienen coordenadas GPS?
SELECT column_name, data_type
FROM information_schema.columns
WHERE table_name IN ('gps_registros','volvo_ecu')
AND column_name ILIKE '%lat%'
   OR column_name ILIKE '%lng%'
   OR column_name ILIKE '%lon%'
   OR column_name ILIKE '%coord%';

-- ¿Cuántos camiones CENCOSUD tienen GPS?
SELECT c.patente, c.id,
  COUNT(g.id) as puntos_gps
FROM camiones c
LEFT JOIN gps_registros g ON g.camion_id = c.id
  AND g.timestamp_gps >= '2026-03-01'
WHERE c.faena ILIKE '%CENCOSUD%'
   OR c.cliente ILIKE '%CENCOSUD%'
GROUP BY c.patente, c.id
ORDER BY puntos_gps DESC;

-- ¿Qué campos de velocidad y km hay?
SELECT column_name FROM information_schema.columns
WHERE table_name = 'gps_registros';

SELECT column_name FROM information_schema.columns
WHERE table_name = 'volvo_ecu';

Usar los resultados reales para construir
las queries correctas con los nombres
de columnas que REALMENTE existen.

MOTOR DE RECONSTRUCCIÓN DE VIAJES
═══════════════════════════════════════════════════

Función reconstruirViajesCamion(camion_id, desde, hasta):

PASO 1 — Obtener puntos GPS reales ordenados:

-- Intentar primero gps_registros
SELECT 
  camion_id,
  latitud as lat,        -- usar nombre real de columna
  longitud as lng,       -- usar nombre real de columna
  timestamp_gps as ts,
  velocidad_kmh,         -- usar nombre real si existe
  km_odometro            -- usar nombre real si existe
FROM gps_registros
WHERE camion_id = $camion_id
AND timestamp_gps BETWEEN $desde AND $hasta
AND latitud IS NOT NULL
AND longitud IS NOT NULL
ORDER BY timestamp_gps ASC;

-- Si gps_registros no tiene coordenadas,
-- intentar volvo_ecu:
SELECT
  camion_id,
  latitud as lat,
  longitud as lng,
  periodo_inicio as ts,
  velocidad_promedio as velocidad_kmh,
  km_odometro
FROM volvo_ecu
WHERE camion_id = $camion_id
AND periodo_inicio BETWEEN $desde AND $hasta
AND latitud IS NOT NULL
ORDER BY periodo_inicio ASC;

Si ninguna tabla tiene lat/lng →
marcar camión como SIN_GPS y no procesar.
Mostrar en UI: "Sin datos GPS disponibles
para este camión en el período."

PASO 2 — Detectar viajes desde los puntos:

Un VIAJE es una secuencia de puntos donde:

INICIO: velocidad > 5 km/h después de estar
detenido > 20 minutos O es el primer punto del día

PARADA INTERMEDIA: velocidad < 5 km/h
durante > 30 minutos continuos
en un punto diferente al origen

FIN: velocidad < 5 km/h durante > 20 minutos
Y el punto está a > 10 km del origen
O no hay más puntos en las próximas 4 horas

Para cada grupo de puntos entre inicio y fin:

km_gps = suma haversine de todos los segmentos
  donde velocidad > 5 km/h
  y gap entre puntos < 15 minutos

duracion_min = ts_fin - ts_inicio en minutos

velocidad_max = MAX(velocidad_kmh) del viaje

velocidad_prom = km_gps / (duracion_min / 60)

tiempo_detenido = suma de minutos donde
  velocidad < 5 km/h dentro del viaje

PASO 3 — Solo guardar viajes con sentido real:

Filtrar y guardar SOLO si:
- km_gps > 30 (viaje real, no movimiento local)
- duracion_min > 60 (mínimo 1 hora)
- duracion_min < 7200 (máximo 5 días)
- tiene al menos 5 puntos GPS

Si no cumple → descartar silenciosamente.

ENDPOINT PRINCIPAL — HISTORIAL POR CAMIÓN
═══════════════════════════════════════════════════

GET /api/geo/camion/:patente/historial
  ?desde=2026-03-01&hasta=hoy

1. Buscar camion_id por patente
2. Verificar si ya hay viajes procesados
   en geo_viajes para ese camión y período
3. Si hay → devolver directamente
4. Si no hay → correr reconstruirViajesCamion()
   y devolver resultado

    patente, contrato, conductor,

    id,
