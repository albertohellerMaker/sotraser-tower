En la pestaña TMS agregar flujo basado en CONTRATOS.
Un contrato = una faena existente en la BD.
El TMS analiza todos los camiones de ese contrato
usando historial Volvo desde 2026-03-01.
NO tocar ninguna otra pestaña.

═══════════════════════════════════════════════════
BASE DE DATOS — agregar tabla contratos TMS
═══════════════════════════════════════════════════

CREATE TABLE IF NOT EXISTS tms_contratos (
  id SERIAL PRIMARY KEY,
  faena_id INTEGER REFERENCES faenas(id),
  nombre VARCHAR(200) NOT NULL,
  cliente VARCHAR(200),
  descripcion TEXT,
  fecha_inicio DATE DEFAULT '2026-03-01',
  activo BOOLEAN DEFAULT true,
  total_viajes INTEGER DEFAULT 0,
  km_total NUMERIC(12,1) DEFAULT 0,
  litros_total NUMERIC(12,2) DEFAULT 0,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS tms_viajes (
  id SERIAL PRIMARY KEY,
  contrato_id INTEGER REFERENCES tms_contratos(id),
  camion_id INTEGER REFERENCES camiones(id),
  codigo VARCHAR(30) UNIQUE NOT NULL,
  conductor VARCHAR(100),
  origen_nombre VARCHAR(200),
  origen_lat NUMERIC(10,7),
  origen_lng NUMERIC(10,7),
  destino_nombre VARCHAR(200),
  destino_lat NUMERIC(10,7),
  destino_lng NUMERIC(10,7),
  fecha_salida TIMESTAMP,
  fecha_llegada TIMESTAMP,
  fecha_cierre TIMESTAMP,
  estado VARCHAR(20) DEFAULT 'DETECTADO',
  -- DETECTADO · CONFIRMADO · EN_RUTA · 
  -- COMPLETADO · CANCELADO
  km_inicio NUMERIC(10,1),
  km_cierre NUMERIC(10,1),
  km_recorridos NUMERIC(10,1),
  litros_sigetra NUMERIC(8,2),
  litros_ecu NUMERIC(8,2),
  diferencia_litros NUMERIC(8,2),
  rendimiento_real NUMERIC(6,2),
  detectado_por_ia BOOLEAN DEFAULT false,
  confirmado BOOLEAN DEFAULT false,
  notas TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);

CREATE TABLE IF NOT EXISTS tms_paradas (
  id SERIAL PRIMARY KEY,
  viaje_id INTEGER REFERENCES tms_viajes(id),
  orden INTEGER NOT NULL,
  nombre VARCHAR(200),
  direccion VARCHAR(300),
  lat NUMERIC(10,7),
  lng NUMERIC(10,7),
  tipo VARCHAR(20) DEFAULT 'ENTREGA',
  -- ENTREGA · CARGA · DESCANSO · COMBUSTIBLE
  estado VARCHAR(20) DEFAULT 'PENDIENTE',
  -- PENDIENTE · EN_CAMINO · COMPLETADO
  hora_estimada TIMESTAMP,
  hora_real TIMESTAMP,
  notas TEXT
);

CREATE TABLE IF NOT EXISTS tms_puntos (
  id SERIAL PRIMARY KEY,
  contrato_id INTEGER REFERENCES tms_contratos(id),
  camion_id INTEGER REFERENCES camiones(id),
  lat NUMERIC(10,7),
  lng NUMERIC(10,7),
  nombre_inferido VARCHAR(200),
  tipo VARCHAR(20),
  veces_visitado INTEGER DEFAULT 1,
  duracion_promedio_min INTEGER,
  primera_visita TIMESTAMP,
  ultima_visita TIMESTAMP,
  confirmado BOOLEAN DEFAULT false
);

ENDPOINTS TMS
═══════════════════════════════════════════════════

GET /api/tms/contratos
  Lista todos los contratos activos con stats

POST /api/tms/contratos
  Crear contrato desde faena existente:
  Body: { faena_id, descripcion }
  Auto-puebla nombre y cliente desde faenas

GET /api/tms/contratos/:id
  Contrato completo con:
  - Datos del contrato
  - Lista de camiones asignados a la faena
  - Stats: viajes, km, litros, rendimiento
  - Viajes activos y recientes

POST /api/tms/contratos/:id/analizar
  ENDPOINT PRINCIPAL — análisis completo:
  1. Obtener todos los camiones de la faena
  2. Para cada camión obtener GPS desde 2026-03-01
  3. Detectar paradas de +30 min por proximidad 500m
  4. Identificar viajes (movimientos entre paradas)
  5. Cruzar con cargas Sigetra del mismo período
  6. Llamar a Claude para inferir nombres y clasificar
  7. Guardar en tms_viajes, tms_paradas, tms_puntos
  8. Devolver resumen del análisis

GET /api/tms/contratos/:id/viajes
  Todos los viajes del contrato
  Query: estado, camion_id, desde, hasta

GET /api/tms/contratos/:id/mapa
  Todos los puntos y rutas del contrato
  para dibujar en el mapa

GET /api/tms/contratos/:id/stats
  KPIs del contrato:
  total viajes, km promedio por viaje,
  rendimiento promedio, litros total,
  mejor y peor camión, ruta más frecuente

POST /api/tms/viajes/:id/confirmar
  Confirmar viaje detectado por IA

POST /api/tms/viajes/:id/cerrar
  Cierre con cuadratura Sigetra + ECU

LÓGICA DE DETECCIÓN DE VIAJES
═══════════════════════════════════════════════════

En POST /api/tms/contratos/:id/analizar:

Para cada camión del contrato:

1. Obtener registros GPS ordenados por fecha:
   SELECT lat, lng, periodo_inicio, km_odometro,
          velocidad_promedio
   FROM volvo_ecu
   WHERE camion_id = X
   AND periodo_inicio >= '2026-03-01'
   ORDER BY periodo_inicio ASC

2. Detectar PARADAS (velocidad < 5 km/h 
   por más de 30 minutos):
   Si tiempo entre registros > 30 min 
   Y distancia < 500m = PARADA

3. Detectar MOVIMIENTOS entre paradas:
   Cada movimiento entre 2 paradas = viaje potencial
   Solo considerar si km recorridos > 20 km

4. Para cada viaje potencial:
   - Buscar cargas Sigetra del camión 
     en ese período
   - Calcular litros ECU del período
   - Guardar en tms_viajes con 
     estado = 'DETECTADO', detectado_por_ia = true

5. Agrupar paradas cercanas (radio 500m):
   Si varios registros caen en radio 500m
   = mismo punto físico
   Contar frecuencia y duración promedio

6. Llamar a Claude para inferir nombres
   de los clusters de paradas
