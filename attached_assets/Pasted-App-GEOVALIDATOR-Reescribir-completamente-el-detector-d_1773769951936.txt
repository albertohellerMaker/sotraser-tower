PASO 0 — ENTENDER LOS DATOS ANTES DE DETECTAR
═══════════════════════════════════════════════════

Al arrancar el servidor ejecutar esta query
y guardar el resultado en memoria:

SELECT 
  c.id as camion_id,
  c.patente,
  COUNT(g.id) as total_puntos,
  MIN(g.timestamp_gps) as desde,
  MAX(g.timestamp_gps) as hasta,
  AVG(g.velocidad_kmh) as vel_promedio,
  SUM(CASE WHEN g.latitud IS NOT NULL 
      AND g.longitud IS NOT NULL 
      THEN 1 ELSE 0 END) as puntos_con_coords,
  MAX(g.km_odometro) - MIN(g.km_odometro) 
    as km_delta_periodo
FROM camiones c
JOIN gps_registros g ON g.camion_id = c.id
WHERE g.timestamp_gps >= '2026-03-01'
AND (c.faena ILIKE '%CENCOSUD%' 
  OR c.cliente ILIKE '%CENCOSUD%')
GROUP BY c.id, c.patente
HAVING COUNT(g.id) > 0
ORDER BY total_puntos DESC;

Si gps_registros no tiene datos →
repetir con volvo_ecu usando periodo_inicio
como timestamp y latitud/longitud si existen.

Usar la tabla que tenga datos reales.
Guardar en variable FUENTE_GPS = 'gps_registros'
o 'volvo_ecu' según cuál tenga datos.

═══════════════════════════════════════════════════
HAVERSINE — función base
═══════════════════════════════════════════════════

function haversineKm(lat1, lon1, lat2, lon2) {
  if (!lat1||!lon1||!lat2||!lon2) return 9999;
  const R = 6371;
  const dLat = (lat2-lat1)*Math.PI/180;
  const dLon = (lon2-lon1)*Math.PI/180;
  const a = 
    Math.sin(dLat/2)**2 +
    Math.cos(lat1*Math.PI/180) *
    Math.cos(lat2*Math.PI/180) *
    Math.sin(dLon/2)**2;
  return R * 2 * Math.asin(Math.sqrt(a));
}

CRITERIOS DE DETECCIÓN — 4 reglas simples
═══════════════════════════════════════════════════

Para que un viaje sea VÁLIDO necesita cumplir
TODOS estos criterios:

CRITERIO 1 — LLEGÓ AL PUNTO (radio 2km):
haversineKm(punto.lat, punto.lng,
            destino.lat, destino.lng) <= 2.0

CRITERIO 2 — ESTABA DESPACIO (velocidad real):
punto.velocidad_kmh <= 15
Si velocidad es null → aceptar igual
(Volvo a veces no reporta velocidad en parada)

CRITERIO 3 — PASÓ TIEMPO AHÍ:
Buscar puntos consecutivos dentro del radio 2km
OPCIÓN A: hay 2+ puntos en el radio → 
  tiempo = ts_ultimo_dentro - ts_primero_dentro
  Aceptar si tiempo >= 30 minutos

OPCIÓN B: hay solo 1 punto en el radio →
  Calcular tiempo estimado:
  distancia_punto_anterior = haversine(
    punto_anterior, punto_en_radio)
  distancia_punto_siguiente = haversine(
    punto_en_radio, punto_siguiente)
  Si distancia_anterior > 50km 
  Y distancia_siguiente > 50km →
  el camión llegó y salió → aceptar
  Si solo una de las dos > 50km → aceptar
  con confianza MEDIA

CRITERIO 4 — VENÍA DE LEJOS (viaje real):
El punto GPS más cercano ANTES de entrar
al radio de 2km debe estar a > 50km
del destino.
Esto elimina movimientos locales.

═══════════════════════════════════════════════════
ALGORITMO PRINCIPAL
═══════════════════════════════════════════════════

Función detectarVisitasACamion(camion_id, desde, hasta):

1. Obtener todos los puntos GPS del camión
   ordenados por timestamp ASC
   WHERE timestamp >= desde
   AND latitud IS NOT NULL
   AND longitud IS NOT NULL

2. Si menos de 3 puntos con coordenadas →
   intentar reconstruir desde km_odometro
   (ver MÉTODO ALTERNATIVO abajo)

3. Para cada punto GPS del camión:
   Para cada destino en tarifas_rutas_maestro
   WHERE contrato = 'CENCOSUD':

   a. Calcular distancia al destino
   b. Si distancia <= 2km:
      → Marcar como CANDIDATO a visita
      → Agregar a grupo de visita actual

4. Agrupar candidatos consecutivos al mismo
   destino en una sola visita:
   Si punto[i] y punto[i+1] ambos dentro
   del radio del mismo destino →
   son parte de la misma visita

5. Para cada grupo (visita candidata):
   Aplicar los 4 criterios.
   Si cumple todos → VISITA CONFIRMADA
   Si cumple 3 → VISITA PROBABLE
   Si cumple menos de 3 → DESCARTAR

6. Para cada VISITA CONFIRMADA o PROBABLE:
   Buscar punto de origen:
   Primer punto GPS del día del camión
   O último punto a > 50km antes de llegar

   Calcular km del viaje:
   Si hay km_odometro en los puntos →
   usar delta odómetro
   Si no → usar haversine origen→destino × 1.15
   (factor 1.15 para aproximar ruta real
   vs línea recta)

7. Guardar en geo_viajes:
   {camion_id, patente, contrato,
    origen_nombre, destino_nombre,
    origen_lat, origen_lng,
    destino_lat, destino_lng,
    fecha_viaje, km_viaje,
    velocidad_en_destino,
    tiempo_en_destino_min,
    confianza: 'ALTA'|'MEDIA',
    puntos_dentro_radio,
    tarifa_id, tarifa_clp}

MÉTODO ALTERNATIVO — SIN COORDENADAS GPS
═══════════════════════════════════════════════════

Si los puntos GPS no tienen lat/lng
pero tienen km_odometro:

Usar cargas_tarjeta como proxy de destino:
Si el camión cargó en un surtidor cuya
dirección geocodificada está dentro de
80km de un destino CENCOSUD →
asumir que ese viaje llegó a ese destino.

SELECT ct.patente, ct.fecha_hora,
  ct.litros, ct.surtidor,
  ct.km as km_sigetra,
  sg.lat as surtidor_lat,
  sg.lng as surtidor_lng
FROM cargas_tarjeta ct
LEFT JOIN surtidores_geo sg 
  ON sg.nombre ILIKE '%' || ct.surtidor || '%'
WHERE ct.patente = $patente
AND ct.fecha_hora >= '2026-03-01'
AND ct.cliente ILIKE '%CENCOSUD%'
ORDER BY ct.fecha_hora ASC;

Para cada carga con coordenadas de surtidor:
Buscar destino CENCOSUD más cercano
(radio 80km para abarcar zonas de entrega).
Si hay match → registrar visita probable.

CONFIANZA — cómo se muestra
═══════════════════════════════════════════════════

🟢 ALTA = 
  - 2+ puntos GPS dentro del radio 2km
  - Velocidad confirmada < 15 km/h
  - Venía de > 50km

🟡 MEDIA = 
  - 1 punto GPS dentro del radio 2km
  - Sin dato de velocidad
  - Contexto sugiere que llegó
    (carga Sigetra en la zona ese día)

⚪ SIN DATOS =
  - No hay GPS con coordenadas
  - Solo inferido desde odómetro o Sigetra

Mostrar siempre el nivel de confianza.
Nunca inventar un viaje sin evidencia.
