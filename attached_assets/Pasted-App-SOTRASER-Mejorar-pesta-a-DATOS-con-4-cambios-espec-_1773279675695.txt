CAMBIO 1 — MICRO-CARGAS: solo sospechosos reales
═══════════════════════════════════════════════════

LÓGICA DE DETECCIÓN — 3 señales de robo confirmadas:

SEÑAL A — Múltiples cargas pequeñas mismo día:
SELECT patente, DATE(fecha_hora) as dia,
  COUNT(*) as cargas_dia,
  SUM(litros) as litros_total,
  COUNT(DISTINCT surtidor) as surtidores_distintos
FROM cargas_tarjeta
WHERE litros < 100
GROUP BY patente, DATE(fecha_hora)
HAVING COUNT(*) >= 2
AND COUNT(DISTINCT surtidor) >= 2

→ Esto es sospechoso porque un camión no necesita
  cargar dos veces pequeñas en distintos lugares
  el mismo día.

SEÑAL B — Carga sin movimiento GPS:
Buscar cargas donde el camión NO tiene registros GPS
en ventana de 4 horas antes Y 4 horas después.
Si el camión no se movió = ¿para qué necesita combustible?

SELECT ct.patente, ct.fecha_hora, ct.litros
FROM cargas_tarjeta ct
LEFT JOIN gps_registros gr ON 
  gr.camion_id = (SELECT id FROM camiones WHERE patente = ct.patente)
  AND gr.timestamp_gps BETWEEN 
    ct.fecha_hora - INTERVAL '4 hours'
    AND ct.fecha_hora + INTERVAL '4 hours'
  AND gr.velocidad_kmh > 5
WHERE gr.id IS NULL
AND ct.litros < 150
AND ct.fecha_hora >= '2026-03-01'

→ Si no hay movimiento registrado = carga fantasma.

SEÑAL C — Carga con tanque casi lleno según ECU:
Buscar en volvo_ecu el nivel de combustible
más cercano al momento de la carga Sigetra.
Si ECU indica >80% de tanque Y Sigetra dice que cargó
= alguien cargó combustible que no entró al tanque.

SELECT ct.patente, ct.fecha_hora, ct.litros,
  ve.nivel_combustible_pct
FROM cargas_tarjeta ct
JOIN camiones c ON c.patente = ct.patente
JOIN volvo_ecu ve ON ve.camion_id = c.id
  AND ABS(EXTRACT(EPOCH FROM (ve.periodo_inicio - ct.fecha_hora))) 
      = (
        SELECT MIN(ABS(EXTRACT(EPOCH FROM (ve2.periodo_inicio - ct.fecha_hora))))
        FROM volvo_ecu ve2 WHERE ve2.camion_id = c.id
      )
WHERE ve.nivel_combustible_pct > 80
AND ct.litros < 150
AND ct.fecha_hora >= '2026-03-01'

Nota: si la columna no se llama nivel_combustible_pct
usar el campo de combustible disponible en volvo_ecu.

CAMBIO 2 — NUEVA TAB: EXCESOS DE VELOCIDAD
═══════════════════════════════════════════════════

Agregar tab [EXCESOS DE VELOCIDAD] en pestaña DATOS.

FUENTE DE DATOS:
SELECT camion_id, periodo_inicio,
  velocidad_maxima, velocidad_promedio,
  COUNT(*) as registros_sobre_limite
FROM volvo_ecu
WHERE velocidad_maxima > 90
AND periodo_inicio >= '2026-03-01'
GROUP BY camion_id, periodo_inicio, 
         velocidad_maxima, velocidad_promedio

Límite de velocidad: 90 km/h (configurable en parametros)

CRUCE DE PESO CON SIGETRA:

Un camión "cumple con el cruce" si:
1. Tiene tara registrada en el sistema
2. Tiene cargas en Sigetra en el período
3. Se puede calcular: 
   peso_total = tara + carga_transportada
   Y peso_total <= peso_maximo_kg

Endpoint GET /api/datos/peso-tara:
Para cada camión con tara registrada:
- Obtener cargas Sigetra del período
- Calcular si el peso estimado cumple límite
- Cruzar con datos de Volvo si disponible
